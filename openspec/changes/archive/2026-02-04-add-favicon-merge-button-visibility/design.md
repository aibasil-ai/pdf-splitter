## Context

專案為單頁 `index.html`，以 React 狀態管理 UI，所有行為與 UI 控制皆在前端本地完成。目前缺少 favicon，且多檔合併模式的下載按鈕在進入頁面時即顯示，與其他模式的「未輸入時隱藏」一致性不足。

## Goals / Non-Goals

**Goals:**
- 新增 favicon 以提升分頁辨識度。
- 多檔合併模式下載按鈕在無檔案時隱藏，有檔案後再顯示。

**Non-Goals:**
- 不改變現有合併或切割的功能流程。
- 不新增後端資源或儲存機制。

## Decisions

- **favicon 以靜態資源引用為主**
  - 作法：在 `index.html` 的 `<head>` 新增 `<link rel="icon" ...>` 指向本地 favicon 檔案。
  - 替代方案：使用 data URI 內嵌。未採用，因為靜態檔案較好維護。

- **多檔合併下載按鈕條件顯示**
  - 作法：以 `mergeFiles.length > 0` 作為顯示條件；無檔案時按鈕不渲染。
  - 替代方案：顯示但禁用。未採用，因為需求希望「一開始隱藏」。

## Risks / Trade-offs

- **favicon 檔案缺失** → 若未提供檔案會造成 404；在提交前確認 favicon 檔案存在。
- **UI 一致性** → 隱藏按鈕需保持其他模式行為一致；避免在無檔案時顯示空白區塊。
