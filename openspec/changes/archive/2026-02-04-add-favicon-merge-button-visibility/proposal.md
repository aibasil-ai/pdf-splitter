## 為什麼

目前頁面缺少瀏覽器分頁圖示（favicon），分頁辨識度不足；同時多檔合併模式的下載按鈕一進入頁面就顯示，與其他模式「先隱藏、輸入後再顯示」的互動不一致，可能造成使用者困惑。

## 變更內容

- 新增網站分頁圖示（favicon），提升分頁辨識度與產品識別。
- 調整多檔合併模式的下載按鈕顯示邏輯：無檔案時隱藏，有檔案後顯示。

## 能力

### 新增能力
- `site-favicon`: 提供網站分頁圖示（favicon）設定與載入行為。

### 修改能力
- `pdf-merge-files`: 調整多檔合併模式的下載按鈕顯示條件與互動行為。

## 影響範圍

- 影響前端 `index.html` 的 `<head>` 與多檔合併 UI 顯示條件。
- 可能需要新增 favicon 資源檔或改用內嵌資料 URI。
